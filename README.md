# Secure Client-Server System (Go)

## Описание

Данный проект демонстрирует современную защищённую клиент-серверную систему на языке Go с использованием:
- **AES-256 (CBC + HMAC)** для шифрования и проверки целостности сообщений
- **ECDSA** для цифровой подписи
- **ECDH (на базе ECDSA)** и **RSA** для обмена ключами и аутентификации
- **Nonce** и **timestamp** для защиты от повторных атак (replay attacks)
- **Мониторинг производительности** и интегральная оценка эффективности алгоритма

## Архитектура

- **Клиент и сервер** реализованы в одном приложении (`cmd/app/main.go`).
- **Обмен ключами** происходит при установлении соединения (ECDH и RSA).
- **Сессионный ключ** используется для симметричного шифрования сообщений (AES-256).
- **Каждое сообщение**:
  - Шифруется (AES-256-CBC)
  - Снабжается HMAC для проверки целостности
  - Подписывается ECDSA и RSA
  - Содержит nonce и timestamp
- **Проверка** на стороне получателя: HMAC, подписи, nonce/timestamp, расшифровка.
- **Вся криптография** реализована в `internal/crypto/`.
- **Метрики** (время операций, длина ключа, эффективность) — в `internal/metrics/`.
- **Тесты** и утилиты — в `testutils/`.
- **Бенчмарки** — в `benchmark/`.

## Запуск

### Сборка
```bash
cd ~/Desktop/Client-server
bash.exe -c "go build ./cmd/app"
```

### Запуск сервера
```bash
bash.exe -c "go run ./cmd/app/main.go --server --addr=:8080"
```

### Запуск клиента
```bash
bash.exe -c "go run ./cmd/app/main.go --client --addr=localhost:8080"
```

### Доступные команды в чате
- `/stats` — вывести статистику безопасности
- `/test` — запустить тесты безопасности и вывести результаты

## Криптографические детали

- **AES-256-CBC**: симметричное шифрование сообщений, ключ длиной 256 бит, IV 16 байт, дополнение PKCS#7
- **HMAC-SHA256**: контроль целостности, сравнение в постоянное время
- **ECDSA**: цифровая подпись для проверки подлинности
- **RSA**: альтернатива для подписи и обмена ключами
- **ECDH**: вычисление общего секрета для сессионного ключа
- **Nonce и timestamp**: защита от повторных атак

## Формула эффективности

Время и параметры операций нормируются и объединяются в интегральный показатель эффективности:

```
E = w1 * T'enc + w2 * T'dec + w3 * K' + w4 * P'attack
```
- T'enc — нормализованное время шифрования
- T'dec — нормализованное время расшифровки
- K' — нормализованная длина ключа
- P'attack — нормализованная вероятность атаки (обратная)
- w1, w2, w3, w4 — веса (по умолчанию 0.25)

## Протокол обмена

1. Клиент и сервер обмениваются публичными ключами (ECDSA/ECDH, RSA)
2. Вычисляется общий секрет (ECDH)
3. Генерируется сессионный ключ (AES-256)
4. Все сообщения шифруются, подписываются, снабжаются HMAC, nonce, timestamp
5. Получатель проверяет подписи, HMAC, nonce/timestamp, расшифровывает сообщение

## Защита от атак
- **Replay-атаки**: nonce и timestamp в каждом сообщении, трекинг использованных nonce
- **Tampering**: HMAC и подписи
- **Timing-атаки**: сравнение HMAC в постоянное время
- **Brute-force**: длинные ключи, контроль вероятности атаки

## Расширяемость
- Легко интегрируется в web, desktop, mobile, микросервисы
- Можно заменить транспорт (TCP/gob → gRPC/HTTPs)
